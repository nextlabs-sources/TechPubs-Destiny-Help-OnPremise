  • Enrolling users, hosts, and groups

    This section explains enrollment and provides instructions for enrolling users, groups, and host groups from LDAP, and users and user groups from Azure Active Directory (Azure AD).

    • About enrollment

      Enrollment is is the process of importing LDAP directory entities, SharePoint user groups, hosts, and other entities into the Control Center database. For Azure AD, only users and groups can be imported into the Control Center database.

    • Entities that can be enrolled

      The Control Center enrollment utilities import information about the underlying information network, including the entities described in this section.

    • About Domain Group enrollment

      A Domain Group contains multiple User, Host, and Group sources, referred to as subdomains, that can be managed together in a single enrollment process.

    • Overview of enrollment types and procedures

      Different types of enrollments require different arguments and use different auxiliary files.

    • About the Enrollment Manager utility

      The Enrollment Manager utility enrolls entities into the Network Information Directory, directly from an LDAP directory, an LDIF file, or Azure AD.

      • How enrollment connections are established

        The Enrollment Manager utility can be installed on any host in the network, as long as the utility can connect to the server where Control Center is running.

      • Before you enroll

        Before enrolling LDAP data, you must run the Control Center installation wizard, and then start Control Center.

      • Using the Enrollment Manager

        The Enrollment Manager is automatically installed on the Control Center host.

Command line arguments for Enrollment Manager

Command

Purpose

 Full Command String
-enroll Enroll entities from an LDAP directory server or a source file enrollmgr -enroll -t <type> -n <domain_name> -s <server> -p <port> -u <username> {-w <passwd>|-ew <encryptpwd>} -a <ad_connection_file> -d <definitionfile> [-f <filterfile>]
Enroll entities from Azure AD enrollmgr -enroll -t AAD -n <domain_name> -s localhost -p <port> -u <username> {-w <passwd>|-ew <encryptpwd>} -a <Azure_AD_connection_file> -d <Azure_AD_definitionfile>

Not all arguments are required for all enrollment types.
-update Update an existing enrollment from an LDAP directory or a source file enrollmgr -update -t <type> -n <domain_name> -s <server> -p <port> -u <username> {-w <passwd>|-ew <encryptpwd>} -a <ad_connection_file> -d <definitionfile> [-f <filterfile>]

Not all arguments are required for all enrollment types.
-sync Synchronize an enrollment enrollmgr -sync -n <domain_name> -s <server> -p <port> -u <username> {-w <passwd>|-ew <encryptpwd>}
Synchronize an enrollment for Azure AD enrollmgr -sync -n <domain> -u <username> {-w <passwd>|-ew <encryptpwd>}
-delete Delete an enrollment enrollmgr -delete -n <domain_name> -s <server> -p <port> -u <username> {-w <passwd>|-ew <encryptpwd>}
-list List enrollments enrollmgr -list -s <server> -p <port> -u <username> {-w <passwd>|-ew <encryptpwd>}
List enrollments for Azure AD enrollmgr -list -s <server> -p <port> -u <username> {-w <passwd>|-ew <encryptpwd>}
-h Display a help screen showing definitions of all valid command line arguments enrollmgr -h

The following table provides an overview of different arguments that can be supplied to establish connections and to reference the location of input files, including connection, definition, filter, and configuration files.

Argument Description
-t Used with the -enroll and -update commands to specify the type of enrollment. Valid values are:
  • DIR: enrolling from an LDAP server
  • PORTAL: enrolling SharePoint group definitions
  • LDIF: enrolling from an LDIF file
  • DOMAINGROUP: enrolling multiple subdomains in a single enrollment (which can include DIR, PORTAL, and LDIF type enrollments)
  • AAD: enrolling from Azure AD

These values are case sensitive, and must be capitalized as shown.
-n <domain_name> or <enrollment_name> The name of the enrollment. Each enrollment needs a unique name, so it can be identified in various management functions, such as confirming, synchronizing, and updating. For User and Host enrollments, this value must be the name of the domain from which entities are being enrolling. Accordingly, domains can only be enrolled once. After a domain is enrolled, it can be updated using the same name to incorporate any changes.

SharePoint or application enrollments do not need to use the domain name; they can use any name. This makes it possible to perform multiple enrollments from the same SharePoint server, as long as the names are unique.

A Domain Group also receives its own domain name, as does each subdomain in the Domain Group. Subdomain enrollment names are defined in the Configuration file.
-g <domaingroup file> The location of the Domain Group configuration file.
-s <server> The name or URL of the host where Control Center is running. For distributed Control Center installations, where Enrollment Manager is installed on a different host, this server should be the host where the Enrollment Manager is installed. If this argument is not present, the default value localhost is used.
-p <port> The port of the Control Center host, which is 8443 unless it was explicitly changed. This argument is optional; if it is not present, the default value 8443 is used.

-u <username>

 The username of a Control Center user with administrator privileges. NextLabs recommends using a placeholder username, dedicated only to enrollment functions, rather than using a live account. This prevents problems that can arise when a live user changes a username or password.

-w <passwd>

Password for the Control Center administrative user specified by -u. NextLabs recommends using a placeholder password, dedicated only to enrollment functions, rather than using a live account.

-ew <encryptpwd>

Encrypted password for the Control Center administrative user specified by -u. You can generate an encrypted password using the mkpassword.bat utility.

-a <connection_file>

The name of the directory connection file, which is required for enrolling or updating with an LDAP server and for enrolling portal sites. To create a connection file, copy one of several provided template or default files that matches the kind of entity being enrolled.

-d <definitionfile>

The name of the definition file, which is required for all enrollments and updates. To create a definition file, copy one of several provided template or default files that matches the kind of entity being enrolled.

-f <filterfile>

(Optional) The name of the LDAP filter file.

-v

Verbose option; may be used with any command. When this is used and an error occurs, a detailed description of the issue appears in the command window. If this option is not used, a shorter summary is displayed.

-delta

(Optional) Make changes to an existing enrollment after performing manual synchronization.
