Registering custom obligations

Custom obligations are those that are not automatically provided in the Control Center interface, but require some special configuration, referred to as registration.

Registering an obligation involves adding an <Obligation> subsection defining it, within the <Obligations> section of the config file.

There are two kinds of custom obligations, executed and delegated. Both must be registered in the configuration file. The following table provides descriptions of the elements required for registration.
Table 1. Defining custom obligations in configuration.xml
Section Element Function
<Obligations> Section of the XML file that contains all registered custom obligations. That is, registering an obligation means adding an <Obligation> sub-section within this section.
<Obligation> Registers one custom obligation, which behaves as defined by the elements in this section. The Obligations section may have any number of Obligation sections within it.
DisplayName The display name of the item. Names cannot be longer than 255 characters, and cannot include the reserved characters $, /, ?, \, &, or *. Names can include multibyte characters, such as those used by Asian languages.
RunAt This specifies where the obligation runs, which also served to distinguish between executed and delegated types. Valid values: PDP (for executed obligations), PEP (for delegated obligations)
RunBy Used by executed obligations only; specifies what kind of session the obligation requires. Valid values: User (an authenticated user session), or System (an unauthenticated session). Used only when RunAt=PDP.
ExecPath Used by executed obligations only; specifies the path where the executable code has been installed on the enforcer host.
<Arguments> Defines all arguments required for this obligation. Each argument is defined within one <Argument> sub-section.
<Argument> Sub-section within <Arguments> that defines one argument for the parent obligation. Definition consists of a name and all valid values. Each value must be specified by a Value element.

Each argument may be designated “usereditable=yes”. When this flag is present, the argument field is open for users to provide any value, in addition to those predefined here. If this flag is absent, the field is not editable, and users can only select one of the defined values.
Name Specifies the name of the argument that is passed in the K-V pair in the command line. This is also the label that appears in the policy editor to identify this argument.
Value Specifies a valid value for this argument. To mark a value as the default—the value that appears when the user selects this argument in an obligation definition panel—include the following syntax:

<Value default=”true”>value</value>
